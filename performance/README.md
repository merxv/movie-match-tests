# Performance Testing

This directory contains the performance-testing assets for Assignment 3.

## High-risk modules under test

1. `GET /api/movies`
   The controller performs `Movie.find()` without paging or filtering, so response time can degrade as the catalog grows.
2. `GET /api/recommend`
   The recommendation controller executes two Neo4j queries and merges the result in memory, which makes it the highest-risk backend path.
3. `POST /api/users/login` + `GET /api/users/profile`
   This flow combines bcrypt password verification, JWT generation, MongoDB populate, and interaction-history lookup.

## Scenario profiles

| Profile | Intent | Virtual users / load shape | Duration |
| --- | --- | --- | --- |
| `normal` | Baseline user traffic | ramp to 10 VUs | 5 min |
| `peak` | Expected busy period | ramp to 40 VUs | 11 min |
| `spike` | Sudden burst | jump to 80 VUs | 3 min |
| `endurance` | Stability over time | constant 15 VUs | 20 min |

## Prerequisites

1. Start MongoDB and Neo4j.
2. Write app env files:
   `npm run ci:write-env -- "C:\Users\legionnaire\Desktop\study\1 trimester\базы данных\ass6ver2"`
3. Start the application backend from `ass6ver2\backend`.
4. Seed deterministic data:
   `npm run ci:seed`
5. Install `k6` locally if it is not already available.

## Execute tests

Run one of the predefined profiles:

```powershell
npm run perf:normal
npm run perf:peak
npm run perf:spike
npm run perf:endurance
```

Optional overrides:

```powershell
$env:BASE_URL = "http://127.0.0.1:4000"
$env:PERF_PROFILE = "peak"
k6 run .\performance\scenarios\api-performance.js
```

## System resource capture

Collect Windows CPU, memory, and I/O metrics in parallel with the k6 run:

```powershell
$env:PERF_DURATION_MINUTES = 20
powershell -ExecutionPolicy Bypass -File .\performance\scripts\collect-system-metrics.ps1
```

## Outputs

- `performance/results/<profile>-summary.json`: raw summarized k6 metrics
- `performance/results/latest-summary.json`: latest run snapshot
- `performance/results/latest-summary.md`: markdown report generated by `npm run perf:report`
- `performance/results/system-metrics-*.csv`: CPU, memory, and disk counters for charts

Render the latest markdown summary:

```powershell
npm run perf:report
```
